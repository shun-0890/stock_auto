# 記事執筆

デイリーリサーチの結果をもとに、note向けの投資分析記事を作成します。
記事は **2種類** あり、それぞれ独立した記事として作成します。

## 共通前提

- 参照ファイル：当日の `reports/YYYY-MM-DD_macro.md`、`reports/YYYY-MM-DD_screening.md`、`reports/YYYY-MM-DD_evaluation.md`、`reports/YYYY-MM-DD_deepdive_XXXX.md`
- 対象読者：テンバガーを本気で狙いに行く個人投資家
- 執筆スタイル：
  - 忖度なし・根拠ベースで断言する（「〜と思われる」は使わない）
  - 専門用語は使うが、必ず根拠・背景を添える
  - 箇条書き・表を積極活用し視認性を高める
  - 煽りや誇張はしない。事実と分析を淡々と、しかし熱量を持って書く

## 記事①：マクロ分析記事

### 概要
当日のマクロ環境を読者と共有し、「今どういう相場環境か」「どのセクターに資金が向かいやすいか」を伝える記事。

### 参照ファイル
- `reports/YYYY-MM-DD_macro.md`

### 構成

**タイトル例（20〜30文字）**
- 「米イラン停戦延長の代償：原油高が強要するポートフォリオ再編 ～YYYY/MM/DD マクロ分析 #XX～」

**本文構成**
1. **本日の一言サマリー**（リード文、100字以内）
   - 読者が最初に読む一文。今日の相場を一言で言い切る。

2. **主要市場の動き**
   - 米国市場・日本市場の主要指数を表形式で記載
   - 前日比と方向感（上昇/下落/横ばい）を明記

3. **為替・コモディティのポイント**
   - ドル円の水準と方向感
   - 原油・金など株式市場に影響する動き

4. **金利・中央銀行動向**
   - 米10年債・日本10年債の水準
   - FRB/日銀の政策スタンスの変化

5. **地政学・マクロリスク**
   - 今週の主要リスクを箇条書き（波及度★付き）

6. **本日の注目テーマ・セクター**
   - 資金が集まっているテーマとその背景
   - 🔥（強い）/ ➡️（中立）/ ❄️（弱い）で評価

7. **総合判断：今日の相場環境**
   - リスクオン🟢 / 中立🟡 / リスクオフ🔴 の判定と根拠
   - 有利なセクター・不利なセクター
   - 個人投資家として今日どう動くべきか

`reports/YYYY-MM-DD_article_macro.md` に保存する。

## 記事②：銘柄スクリーニング記事

### 概要
スクリーニング・評価・詳細調査の結果を統合し、「今日のテンバガー候補はどれか」を伝える記事。

### 参照ファイル
- `reports/YYYY-MM-DD_screening.md`
- `reports/YYYY-MM-DD_evaluation.md`
- `reports/YYYY-MM-DD_deepdive_XXXX.md`（詳細調査実施分）

### 構成

**タイトル例（20〜30文字）**
- 「クラウド・IT・環境関連が躍動、小型株に資金集中 ～YYYY/MM/DD 本日のテンバガー候補銘柄選定 #XX～」

**本文構成**
1. **本日の一言サマリー**（リード文、150字以内）
   - スクリーニング結果の全体感と今日の狙い目を一言で。

2. **スクリーニング概要**
～選定の流れ～
◆ベース
GeminiのカスタムGemを使用し、選定・調査・分析を実施。最終的に候補として残す、エントリーする銘柄は私が判断します。
流れとしては以下のようになります。
※しばらくは1はGensparkも使用

銘柄スクリーニング実施（最大で10銘柄選定）

1で選定した銘柄の構造的成長ストーリーランク付け

2で私が気になった銘柄の詳細情報調査・分析

※2の構造的成長ストーリーについては以下記事をご覧ください。
https://note.com/takuminatoushi/n/n63c9cb0d5219

3. **銘柄スクリーニング結果**
   - 銘柄コード・銘柄名・時価総額・株価上昇要因・選出理由を表形式で

4. **評価ハイライト**
   - S/A判定銘柄を重点紹介（評価軸ごとの強み・弱みを端的に）
   - B/C判定銘柄は見送り理由を一言添える

5. **詳細調査：注目銘柄の深掘り**（詳細調査実施分のみ、銘柄ごとに）
   - 構造的成長ストーリーの核心
   - 財務・成長ドライバーのポイント
   - リスクと投資判断（買い推奨 Yes/No・目標株価・損切りライン）
   - 詳細分析まとめ（要点を10行程度で）

6. **本日のアクションプラン**
   - 今日・今週すべき具体的アクション（買い・ウォッチ・見送り）
   - ポジション構成への示唆があれば記載

## note への下書き投稿

記事ファイルを保存した後、以下のコマンドを実行して note.com に下書きとして投稿する。

```bash
python scripts/post_to_note.py reports/YYYY-MM-DD_article_macro.md
python scripts/post_to_note.py reports/YYYY-MM-DD_article_screening.md
```

**初回のみ：** プロジェクトルートに `.env` ファイルを作成し、note.com の認証情報を設定する。
```
NOTE_EMAIL=your_email@example.com
NOTE_PASSWORD=your_password
```

投稿後に下書きURLが表示されるので、note.com の編集画面で最終確認してから公開する。
